# Proyecto de Laboratorio de Hacking Ético para Máster UCM Ciberseguridad

## Intención

Crear un repositorio de laboratorios prácticos de hacking ético para el Máster Oficial en Ciberseguridad: Seguridad Defensiva y Ofensiva de la UCM. Los laboratorios están diseñados para proporcionar experiencia práctica en técnicas modernas de ciberseguridad usando contenedores Docker para una fácil configuración y aislamiento.

## Alcance

### Objetivos Principales

1. **Inicialización del Proyecto**: Establecer la estructura base del repositorio
2. **Creación de Laboratorios Docker**: Desarrollar 8 unidades de laboratorio con contenedores Docker
3. **Documentación Completa**: Proporcionar guías de instalación, solución de problemas y documentación técnica
4. **Procesamiento SDD**: Implementar la metodología Spec-Driven Development

### Laboratorios a Incluir

1. **Unidad 1 - Fundamentos y Reconocimiento Agéntico**
2. **Unidad 2 - Vulnerabilidades en IA y Modelos de Lenguaje**
3. **Unidad 3 - Explotación Web y APIs 2026**
4. **Unidad 4 - Hacking de Identidad y AD Moderno**
5. **Unidad 5 - Pentesting Autónomo y Red Teaming Agéntico**
6. **Unidad 6 - Evasión de Defensas**
7. **Unidad 7 - Ciberseguridad Industrial**
8. **Unidad 8 - Post-Explotación, Reporte y Ética**

## Enfoque

### Tecnologías Utilizadas

- **Docker**: Containerización de laboratorios
- **Kali Linux**: Distribución de pentesting
- **Git**: Control de versiones
- **Markdown**: Documentación

### Metodología de Desarrollo

Seguiremos la metodología Spec-Driven Development (SDD) con las siguientes etapas:
1. **initiate**: Inicialización del proyecto y recolección de requisitos
2. **propose**: Propuestas de cambio y definición de alcance
3. **spec**: Especificaciones detalladas y criterios de aceptación
4. **design**: Arquitectura técnica y diseño de implementación
5. **tasks**: Desglose de tareas y checklist de implementación
6. **apply**: Implementación y cambios de código
7. **verify**: Verificación y pruebas
8. **archive**: Archivado de documentación y control de versiones

## Beneficios

1. **Facilidad de Uso**: Laboratorios containerizados para una rápida configuración
2. **Aislamiento**: Cada laboratorio se ejecuta en un entorno seguro y aislado
3. **Actualización**: Fácil mantenimiento y actualización de contenido
4. **Escalabilidad**: Posibilidad de agregar nuevas unidades de laboratorio
5. **Consistencia**: Entornos uniformes para todos los estudiantes

## Riesgos

### Riesgo 1: Dependencia de Imágenes Docker Oficiales
- **Mitigación**: Usar solo imágenes oficiales de fuentes confiables (Kali, OWASP)
- **Contingencia**: Crear copias locales de imágenes críticas

### Riesgo 2: Rendimiento de Contenedores
- **Mitigación**: Recomendar recursos mínimos y optimizar contenedores
- **Contingencia**: Proporcionar versiones ligeras de laboratorios

### Riesgo 3: Compatibilidad entre Plataformas
- **Mitigación**: Probar contenedores en diferentes sistemas operativos
- **Contingencia**: Proporcionar guías específicas para macOS, Linux y Windows

## Conclusión

Este proyecto proporcionará una plataforma de laboratorio integral para estudiantes del Máster UCM Ciberseguridad, con un enfoque práctico y actualizado en técnicas de hacking ético modernas.
